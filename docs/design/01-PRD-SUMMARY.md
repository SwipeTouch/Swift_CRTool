# PRD Summary — Swipetouch CRMS

## Problem

Customization and support requests are fragmented (osTicket). Schools/colleges need structured submission, approval, assignment, tracking, and analytics.

## MVP scope (Phase 1)

1. **Organization onboarding** — Admin creates school/college profiles and default approver
2. **User management** — Client users, approvers, customer-success staff
3. **Change request workflow** — Submit → approve → assign → progress → resolve → close
4. **Comments** — Internal (staff only) vs client-visible (shown only to CR raiser / org clients)
5. **External tickets** — Link JIRA or osTicket reference on a CR
6. **Dashboards** — Role-specific home views
7. **Reporting** — Per-client CR lists, avg time to close, delays, top modules

## Out of scope (later)

- File attachments, WhatsApp, AI categorization, multi-level approval chains, billing
